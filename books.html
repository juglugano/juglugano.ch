  • Programming Android, 2nd Edition

    Zigurd Mednieks, Laird Dornin, G. Blake Meike, Masumi Nakamura

    ISBN: 978-1449316648 - O'Reilly Media Reviewer: Marco Mojana Started: 26-11-2012 Finished on: 06-02-2013 More
    Keywords

    java, android, mobile


    Review:

    The book initially guides you step-by-step in the setup of the development environment on Linux/MacOS/Windows giving important tips about how to solve platform-specific issues to ensure a quick start also for less skilled users.
    I have greatly appreciated that in the first part of the book the basic concepts are explained in a meaningful order and without all the details, leaving the developer with the taste of the choices that have brought the Android platform to life. On the contrary, the official Android documentation, despite being well made, follows the natural way of its underlying support, the web: every topic is tackled separately but the cross-references could easily confuse the novice.
    After the initial overview, the essential components of an Android application are dissected using working code snippets and illustrations. It has been very useful for me that the authors have inserted on purpose some errors in the code to reproduce the most common and hard to spot mistakes every Android developer will encounter on his/her path.
    Speaking of weak points, there are two choices that left me slightly disappointed. Also if Java for Android is not directly related to the Sun implementation, from a practical point of view the language syntax perfectly matches and the APIs contain the same classes with the same methods, so I think that it's a nonsense to spend an entire chapter trying to teach the basics of Java: it is too short for somebody that has never used it and useless for everybody else. I was also surprised that there is no mention of libraries that can simplify the development, e.g. AndroidAnnotations.
    All in all, this book covers in great detail and with simple examples every aspect an Android developer could be interested in, guiding him/her building a well-designed, modular, reusable and robust application.

  • Presentation Zen: Simple Ideas on Presentation Design and Delivery

    Garr Reynolds

    ISBN: 978-0321525659 - New Riders Press Reviewer: Sebastiano Cobianco Started: 11-10-2011 Finished on: 20-01-2012 More
    Keywords

    slides, powerpoint, communication


    Review:

    When I first saw a zero words presentation, a couple of years ago, it took me a while before i could recover from the positive shock it made on me. How comes that someone can stand up for 30 minutes and talk business with no bullet point slides? And be entertaining and connected with the audience instead of boring and pointless?

    Then it was time for Steve Jobs keynote speech on iPhone...yesss, that was a show! :)

    These emotions came back suddenly when a colleague suggested me to have a look at

  • Programming Concurrency on the JVM: Mastering Synchronization, STM, and Actors

    Venkat Subramaniam

    ISBN: 978-1-93435-676-0 - the pragmatic bookshelf Reviewer: Andrea Chiodoni Started: 21-09-2011 Finished on: 19-10-2011 More
    Keywords


    Review:

    Honestly speaking I don't consider myself an expert concurrent programming developer. With Programming Concurrency on the JVM, Venkat was able to clearly explain me the difficulties of the traditional (shared mutability) Java concurrency programming model and provide me alternatives programming models and design techniques (basically by promoting isolated mutability and pure immutability).

    The book is fairly practical and easy to read. Code samples accurately compare the ""performance"" of the different solutions. I appreciated the choice of the author to keep the same samples all along the book and to solve them with the different approaches. It made direct comparisons between those approaches easy.

    A very pleasant surprise has been to discover how the book is up-to-date: Java 7 Fork-Join API is explained on chapter 4.

    When I started reading the book I didn't pay attention to the tittle and I was expecting a Java book. However all the samples are also presented in Scala offering the reader a possibility to compare Java vs. Scala code. Beside Java and Scala, chapters 7 and 9 show how to deal with concurrency on Groovy and JRuby.

    Actor based samples are mainly done using Akka (GPars is introduced for the groovy language). Two Software Transactional Memory are explained: Clojure STM and Akka/Multiverse STM.

    The book is a highly recommended compendium of Java Concurrency in Practice.

    Well done Venkat!


  • Effective Java (2nd Edition)

    Joshua Bloch

    ISBN: 978-0321356680 - Prentice Hall Reviewer: Lauro Canonica Started: 21-03-2011 Finished on: 20-05-2011 More
    Keywords

    java best practices patterns


    Review:

    I have many years of Java development behind me so when people recommended this book I thought: Uff, one more book about Java. How is it possible to write an interesting book after all the already existing Java books? Then I started reading random sections and I had to change my mind. The book is written for experienced Java developers. It won’t explain the basics of programming in Java (nothing for newbies) but it concerns itself with the real issues of developers.

    The book is divided in 78 items and, for every topic, it comes with clear explanations and recommended solutions. The explanations are mostly concise and to the point (only a few times did I have to skip some pages of, in my opinion, theoretical discussions).
    This book is actually a second edition (revised for Java 6) and was started for Java 5. So particular attention is given to the new features of Java 5 such as Generics, Autoboxing, Enums and Annotations.

    If you are wondering if you should read the book just check if you know the answer to the following simple questions.
    • What are the advantages of a static factory over a constructor?
    • What’s wrong with the statement String s= new String(“hello”); ?
    • What are the dangers of Autoboxing?
    • What are the dangers of overriding equals?
    • How do I eliminate unchecked warnings?
    • When are Lists better than Arrays?
    • What’s the problem with float and double?
    • Why is implementing serializable a delicate issue?

    If you know all the answers then you still might want to read it for a clear and precise explanation.

    The fact that the book doesn’t have to be read from cover to cover following a strict order makes it a book that you can pick when you have 10 minutes to spare and just read one item at the time. That’s, at least, how I read it and it improved or reassured my knowledge of the Java Language.

    So I’m recommending the book and I say: Good job Joshua!

But I was missing a complete overview of the system and I was not able to optimize my application architecture in order to get advantage of the powerful features implemented in GAE.

  • Cassandra: The Definitive Guide

    Eben Hewitt

    ISBN: 978-1-4493-9041-9 - O'Reilly Media Reviewer: Roberto Bentivoglio Started: 07-12-2010 Finished on: 22-02-2011 More
    Keywords

    DB, NoSQL, Cassandra


    Review:

    Cassandra is one of the most famous NoSQL database. It was created by Facebook and it was open-sourced in July 2008. Cassandra was accepted into the Apache Incubator and from March 2010 is one of the top project.
    In the same period I started to develop a project based upon the Cassandra db.
    One of the major drawbacks was the total lack of a book dedicated to it while the official
    documentation wasn’t so accurated.
    “Cassandra - The Definitive Guide” is written by Eben Hewitt. He is the Director of Application Architecture at a global hotelier where he focus on event-driven SOA, BPM, mobile, and big data solutions to support them. He wrote also “Java SOA Cookbook”.
    “Cassandra - The Definitive Guide” was released in the November 2010 and is what I’d like to have when I started to work on Cassandra.
    This book can be useful both for people that want to know more about the topic and for people with a good background on it.
    Some chapters as “Introducing Cassandra” or “Installing Cassandra” explain what Cassandra is and how successfully install it. This chapters are useful for the Cassandra newbie.
    Other chapters as “The Cassandra Data Model” are very important because they help to understand the difference respect to an E/R data model. Some chapters are about the internal structure of the project (as “The Cassandra Architecture”) while others are really useful for the production deploy and maintenance of the application and the db (“Monitoring”, “Maintenance” or “Performance Tuning”).
    “Cassandra - The Definitive Guide” is a good book and I feel to suggest it to everyone that want to know more about the argument but also to the developer and the system administrator that yet work on it.


  • REST in Practice

    Jim Webber, Savas Parastatidis, Ian Robinson

    ISBN: 978-0-596-80582-1 - O'Reilly Media Reviewer: Andrea Polci Started: 07-12-2010 Finished on: 17-02-2011 More
    Keywords

    REST


    Review:

    When I started reading “REST in Practice” looking for an in depth book on “what” REST is, “when” it's a viable solution and “why” one should use it. I was less interested in “how” to implement a REST service.

    I'm really satisfied on what I found.

    The book use the development of an example service to guide the reader throw the different aspects of REST without spending too much time in the implementation details, but concentrating on the understanding of the concepts.

    The examples in the different chapters use different tools and languages, ensuring the reader don't get lost in the details of the tools but remain focused on the architecture, with the help of concrete examples. Instead if one is looking for a comprehensive guide to a specifict tool he'll have to look elsewere (like “RESTful Java – with JAX-RS” from Bill Burke, O'Really).
    As a downside I have to admit that it was hard to resist the temptation of skipping examples using uninteresting (to me) technologies (all the .NET stuff).

    In general I can say that the book is really clear and I found it's level appropriate to guide the reader through the comprehension of the different arguments.

    There is just one argument that I'd like the book have spent more time on: transactions. It's treated in a small section near the end of the book and I have to admit that I still don't have a clear understanding of it. I have also to say that it isn't a bad thing if a book leaves the reader with the desire of further studying on some aspects.

    I suggest this reading to all that are interested in the REST architecture or in the development of web services in general.

  • Search Patterns: Design for Discovery

    Peter Morville, Jeffery Callender

    ISBN: 0596802277, 978-0596802271 - O'Reilly Media, Inc., 2010 Reviewer: Luca Manno Started: 20-04-2010 Finished on: 20-10-2010 More
    Keywords

    Computers / Internet / Web Site Directories Computers / User Interfaces Technology & Engineering / Engineering (General)


    Review:

    Search is a critical part of every application. But, what is ‘search’ today? Behind this simple term there is a lot of work and study to do to offer users the best experience. Nowadays search doesn’t mean only keyword finding.
    Peter Morville and Jeffery Callender investigate many fields such as psychology, interaction design, information architecture and technology in order to show us what are the right patterns we should apply when developing and designing for search.
    The main suggestion authors give to the reader is to let the user explore while searching. Starting for a simple keyword term and then refining results while iterating.

Each pattern is analyzed in different contexts (web, mobile...).
    Following these paths helps designers and developers offer the right tools to users to live the best experience of search while achieving their main goal: find useful and meaningful informations.
    Particularly interesting is the last part of the book where the authors explore how search will be in future, for example, when there will be a wide adoption of advanced mobile devices with features like augmented reality and virtuality.

  • The Art of SEO

    Eric Enge, Stephan Spencer, Jessie C. Stricchiola, Rand Fishkin

    ISBN: 0596518862 - O'Reilly Media, Inc., 2009 Reviewer: Mario Fusco Started: 20-04-2010 Finished on: 04-10-2010 More
    Keywords

    Internet marketing Internet searching Program transformation (Computer programming) Web search engines Web site development Web sites


    Review:

    Before start reading this book, I was almost convinced that black magic had more clear and predictable rules than search engine optimization. It happened to me lots of times to read a blog post suggesting a given strategy in order to improve the search ranking of a web site and then to find another post, from an equally authoritative author, giving exactly the opposite advice. Are URLs containing keywords important or not influential to the eyes of a search engine? To repeat the same keyword more and more times in the same page is positive or detrimental? You can find people saying everything and its exact contrary when speaking about these topics.

    This book analyzes these and many more controversial arguments by clarifying how search engines works on their respect and then suggesting the best actions to take in order to make your web site more appealing to them. Indeed, in my opinion, this is one of the most valuable characteristic of the book: it explains the algorithms implemented by people who write the search engines themselves. That makes the set of the suggestions it gives not another list of ingredients to be added to a mysterious magic formula, but, more practically, just a list of viable advices justified by well stated technical reasons.

    If I should make a compliant about the book it is that in some cases it sounds a bit redundant. The same concepts are repeated here and there with tons of sometimes trivial and avoidable examples. I admit too many examples is better than too few, and probably this choice has been made with the purpose to make the book accessible by people with very heterogeneous technical skills. However I believe that to cut some not strictly useful parts could make the book a bit more fluent and enjoyable.

    In the end I recommend this book to everybody interested in search engine optimization. The black art of SEO is a bit less obscure now.

  • Programming Scala: Tackle Multi-Core Complexity on the Java Virtual Machine (Pragmatic Programmers)

    Venkat Subramaniam

    ISBN: 978-1934356319 - Pragmatic Bookshelf; 1 edition (July 1, 2009) Reviewer: Gianfranco Tognana Started: 15-07-2010 Finished on: 09-09-2010 More
    Keywords

    scala, functional programmin, actors, multithread


    Review:

    This book is a fast paced and 'pragmatic' introduction to Scala, a new programming language that mixes functional and OO concepts.
    It is for Java programmers who want to take the adventure to learn this interesting language.

    The book is structured as a single programming exercise to create a multi-threaded application.
    The first two chapters provide an overview of the purpose to be achieved also by presenting snippets of code unreadable for those who have yet to learn the language.
    At this point the reader have already learned an important lesson: Scala is not Java. It integrates quite seamlessly with java and the code runs on the JVM but definitely it is not Java.

    Who will stop here, have the impression that Scala is only a kind of typed version of something similar to javascript.
    Who continues finds that each of the other chapters deals with a part of the initial problem using specific Scala constructs, and any new syntax is explained very clearly.

    All the pieces fit with one another until the presentation of the argument that gives the title: multi core complexity, i.e. multi-threading.
    It turns out that the mechanism for the management of competition in Scala is borrowed from functional languages (especially Haskell), and is based on immutable objects eliminating the race conditions at the root.

    A chapter is devoted to how to test Scala programs with a quick introduction to the available frameworks.

    At the end of the book the application is build with the components developed along the way and at last the first two chapters make sense. Better late than never!

    The code that accompanies the book contains all the examples presented and can be used easily. A note: Some examples include programming errors, the same errors are present in the corresponding source code which obviously does not compile.

    This book in a good introduction to the basics of Scala and could convince the skeptical to give it a try.

    mobile, design, web


    Review:

    First of all, let's clear the field from a possible misunderstanding: this book is not about general mobile design and development but it is about web mobile development. The author states a precise, despite arguable, opinion that brutally said is: do not code native applications but prefer as much as you can web applications. This statement is largely discussed across the book and everyone can make up his own opinion about this. Mine is that currently times are not mature to consider to write just web applications both because mobile browser are not powerful enough (on average) to assure a smooth experience on all devices and because of the lack of a good way to make money from your web app.
    The first three chapters of the book are a really good introduction to the history of mobile, to the mobile current status and to the reasons that should drive an approach to the mobile development. These chapters are a well written recap of the status of the art and present a lot of data useful to understand the global situation. Unfortunately the book is printed in black and white and several pie-charts and graphs are pretty much impossible to read.

An entire chapter is devoted to iPhone web applications development even though most of the concepts presented here are common to other modern devices.
    This is overall a decent introductory book, if you are completely new to the field, and it's packed with many good advices but do not expect much from the practical techniques promised by the title.

If you consider this as a promise, well, this book honors the oath.

When reading this book you won't be overwhelmed by tons of lines of code: examples are short and yet effective.

    Let's go quickly through the chapters:

    The first chapter is a brief history of Hadoop project illustrating its main characteristics and comparing them to those of others similar technologies. Chapter two is a pleasant introduction to MapReduce. The third chapter breaks the continuity of the previous one examining the Hadoop Distributed File System (HDFS subproject) in detail. Chapter four makes a step down in the abstraction layer talking about the Hadoop I/O fundamentals: data integrity, compression, serialization and data structures, explaining the design choice.

    Chapters five to eight are an excellent source for learning Hadoop MapReduce in depth. They cover all the aspects of it: starting from practical ones, such as how to configure, run, test and debug map reduce programs, to those more advanced and formal, like programming models, data formats, sorting and joining tools.

    The two following chapters list few very interesting and useful suggestions for managing and setting up a Hadoop cluster, a precious resource for administrators.

    Chapters eleven to thirteen are for Pig, HBase and Zookeper subprojects under the Hadoop umbrella. Despite of suffering from brevity, they are still interesting.

    Chapter fourteen is made for the reader not to feel alone: important case studies using Hadoop (e.g. Yahoo, and others contributions from Apache Hadoop community).

    My final opinion is that "Hadoop: The Definitive Guide" is a very useful resource for those who want to learn how to ride the "pachydermic" Hadoop (like a "Mahout", perhaps?).

  • Clean Code: A Handbook of Agile Software Craftsmanship

    Robert C. Martin

    ISBN: 978-0132350884 - Prentice Hall, 2008 Reviewer: Mario Fusco Started: 24-09-2009 Finished on: 20-11-2009 More
    Keywords

    agile software development, vertical formatting, implementation patterns, String Arguments, Successive Refinement Listing, Kent Beck


    Review:

    Programmers spend probably more than 80% of their time reading someone else code instead of writing their own. This is why is important that who writes the code pays particular attention not only to what it does but also to how it does that. For example, for a compiler doesn't make any difference if a variable is called p or pageCounter, but of course it makes a big difference for the programmer who have to figure out what kind of information that variable contains. That explains why it is easier to write code for compilers than for people.

    Of course a well chosen name is definitively not enough to make your code readable. This book provides some really good principles that all developers can benefit from using in order to make your code more readable and consequently cheaper to be maintained and easier to be refactored. Among the suggestions made by the book to achieve these results you can find:

    * keep each method as short as possible: 15 lines of code is a reasonable limit that should not be overcome;
    * give each method a single responsibility: if you are trying to give a meaningful name to the method and it comes out that the name should contain an "and" there is a big likelihood that you are breaking this rule;
    * never declare a method with more than 3 parameters: if you need more it is always a good idea to group them as properties of an object;
    * don't use nested cycles or conditions: you can improve both readability and reusability by putting them in little separated methods;
    * write comments only when strictly necessary and keep them updated with the code: there is nothing more useless than a comment that explains what you can easily read from the code or more confounding than a comment that says something different from what the code actually does;
    * fix a set of shared coding standards: a programmer can understand a piece of code faster if he doesn’t meet unexpected surprises while reading it.

    Reading this book you will learn how to implement in your code these and other advices through lots of wonderfully clear and easily reusable examples. The book put an important accent also on how to make your code more robust by building a set of meaningful unit tests and how to leverage that test suite in order to safely improve your code through successive refinements.

    In the end this book will teach you how to make your code easily readable by another programmer, making his job more simple. And this is a valuable thing especially considering that there is a high possibility that the next programmer who will read that code could be yourself.
